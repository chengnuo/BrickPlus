  • BrickPlus 表单会有标准的表单组件包裹结构，可以参考代码演示，这样能最好的呈现出表单的排列和控制表单的各种组合，但这种结构嵌套并不是必须的，也可以单独使用。
实例
                        
<div class="form-group">
    <label class="form-control-label" for="...">
        <b></b> //用作标注必填项
        <span>...</span> //填写表单标题
        <em>：</em> //标题标点符号
    </label>
    <div class="form-control-wrap"> //表单组件定位嵌套
        ...
    </div>
</div>
Keyin Select 模拟 Select 日期选择 模态窗表单 单选和多选 静态数据展示

Keyin

  • 单独的表单控件会被自动赋予一些全局样式。所有设置了 .form-control 类的 <input type="text / password"> 元素都将被默认设置宽度属性为 width: 100%;。
实例
                        
<div class="form-group">
    <label class="form-control-label" for="...">
        <b></b>
        <span>...</span>
        <em>：</em>
    </label>
    <div class="form-control-wrap">
        <input type="text / password / email / tel" class="form-control" id="..." placeholder="..."/>
    </div>
</div>

Select

  • 同样，.form-control 类同样可以影响 select 元素，包括其中的几种展示模式。
实例
                        
<div class="form-group">
    <label class="form-control-label" for="...">
        <b></b>
        <span>...</span>
        <em>：</em>
    </label>
    <div class="form-control-wrap">
        <select class="form-control"/>
        </select>
    </div>
</div>

模拟 Select

  • 模拟 Select 表单组件有固定的 Dom 结构，详细请参考代码演示。
  • 注意 IE7 及以下版本 IE 浏览器的兼容，这里的解决方法是分别用人工给出现在页面的 模拟 Select 和 日期下拉 的 .form-group 层添加内联的 z-index 属性，先出现的层级最大，依次累减。
